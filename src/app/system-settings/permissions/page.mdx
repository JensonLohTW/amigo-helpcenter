# 權限管理

權限管理是確保系統安全和數據保護的核心功能。透過適當的權限設定，您可以控制不同用戶對系統功能和數據的訪問權限，確保營運安全和資訊保密。本指南將協助您掌握權限管理的各項功能。

## 功能概述

權限管理系統提供以下主要功能：

- **角色管理**：創建和管理不同的用戶角色
- **權限分配**：為角色分配適當的系統權限
- **用戶管理**：管理用戶帳戶和角色指派
- **權限審核**：監控和審核權限使用情況
- **安全控制**：設定安全策略和訪問控制
- **權限報表**：生成權限管理相關報表

### 自訂角色創建

#### 創建新角色
1. 進入「系統設定」→「權限管理」
2. 點擊「角色管理」頁籤
3. 點擊「新增角色」
4. 填寫角色資訊：
   - **角色名稱**：清楚描述角色用途
   - **角色描述**：詳細說明角色職責
   - **角色類型**：管理角色 / 營運角色 / 查看角色
   - **有效期限**：角色的有效時間範圍

#### 角色權限設定
**功能模組權限**
- 設備管理：查看 / 編輯 / 刪除
- 商品管理：查看 / 編輯 / 刪除
- 訂單管理：查看 / 處理 / 退款
- 財務報表：查看 / 匯出 / 分析
- 系統設定：查看 / 編輯 / 管理

**數據權限範圍**
- 全部數據：所有設備和數據
- 區域數據：特定區域的數據
- 設備數據：特定設備的數據
- 個人數據：僅自己相關的數據

## 用戶管理

### 用戶帳戶管理

#### 新增用戶
1. 進入「用戶管理」頁籤
2. 點擊「新增用戶」
3. 填寫用戶資訊：
   - **用戶名稱**：系統登入帳號
   - **真實姓名**：用戶真實姓名
   - **電子郵件**：用戶聯絡郵箱
   - **手機號碼**：用戶聯絡電話
   - **部門**：所屬部門
   - **職位**：職位名稱
4. 指派角色和權限
5. 設定帳戶狀態和有效期

#### 用戶狀態管理
**帳戶狀態類型**
- **啟用**：正常使用狀態
- **停用**：暫時停止使用
- **鎖定**：因安全原因鎖定
- **過期**：帳戶已過有效期

**狀態變更操作**
- 啟用帳戶：恢復用戶訪問權限
- 停用帳戶：暫時禁止用戶登入
- 鎖定帳戶：因安全問題強制鎖定
- 重設密碼：強制用戶更改密碼

### 角色指派管理

#### 用戶角色指派
1. 選擇目標用戶
2. 點擊「編輯角色」
3. 選擇適當的角色：
   - 主要角色：用戶的主要職責角色
   - 次要角色：額外的權限角色
   - 臨時角色：有時間限制的特殊權限
4. 設定角色生效時間
5. 儲存角色指派

#### 批量角色管理
**批量指派角色**
- 選擇多個用戶
- 指派相同角色
- 設定統一的有效期限
- 批量確認指派

**角色變更通知**
- 自動發送角色變更通知
- 說明新權限範圍
- 提供使用指導
- 記錄變更歷史

### 權限使用監控

#### 訪問日誌記錄
系統自動記錄以下訪問活動：
- **登入記錄**：用戶登入時間和地點
- **功能使用**：使用的功能模組和操作
- **數據訪問**：查看的數據範圍和內容

#### 異常行為檢測
**自動檢測異常**
- 非正常時間訪問
- 超出權限範圍的操作嘗試
- 大量數據下載
- 頻繁的權限變更

**異常處理流程**
1. 系統自動檢測異常行為
2. 立即發送警報通知
3. 暫時限制相關權限
4. 記錄詳細的異常資訊
5. 通知安全管理員處理

**季度權限檢討**

## 安全控制

### 訪問控制策略

#### IP位址控制
**允許清單**
- 辦公室IP位址範圍
- VPN連接IP位址
- 授權的遠端訪問IP

**禁止清單**
- 已知的惡意IP位址
- 異常訪問來源IP
- 地理位置限制IP

#### 時間訪問控制
**工作時間限制**
- 一般用戶：週一至週五 09:00-18:00
- 管理人員：週一至週六 08:00-20:00
- 系統管理員：24小時訪問權限
- 緊急情況：臨時授權機制

### 多重驗證

#### 雙重驗證設定
**強制雙重驗證角色**
- 系統管理員：必須啟用
- 財務主管：必須啟用
- 營運經理：建議啟用
- 一般用戶：可選啟用

**驗證方式選擇**
- 簡訊驗證碼
- 郵件驗證碼
- 驗證器APP
- 硬體安全金鑰

#### 特權操作驗證
**需要額外驗證的操作**
- 權限變更操作
- 敏感數據匯出
- 系統設定修改
- 大額財務操作

## 權限管理最佳實務

### 權限設計原則

#### 最小權限原則
- 僅授予完成工作所需的最小權限
- 定期檢討和調整權限範圍
- 避免過度授權
- 及時回收不需要的權限

#### 職責分離原則
- 關鍵操作需要多人參與
- 避免單一用戶擁有過多權限
- 建立相互制衡機制
- 重要決策需要審批流程

### 權限管理流程

#### 權限申請流程
1. **提出申請**：用戶提出權限申請
2. **主管審核**：直屬主管審核必要性
3. **安全評估**：安全管理員評估風險
4. **權限授予**：系統管理員執行授權
5. **使用監控**：持續監控權限使用

#### 權限回收流程
1. **定期檢討**：定期檢討權限必要性
2. **識別冗餘**：識別不再需要的權限
3. **通知用戶**：提前通知權限變更
4. **執行回收**：執行權限回收操作
5. **確認生效**：確認權限變更生效

## 常見問題

### Q: 如何申請額外的系統權限？

**A:** 權限申請流程如下：
1. 向直屬主管提出權限申請
2. 說明需要權限的具體原因和用途
3. 填寫權限申請表單
4. 等待主管和安全管理員審核
5. 獲得批准後由系統管理員執行授權

### Q: 忘記自己有哪些權限怎麼辦？

**A:** 您可以：
1. 登入系統後查看「我的權限」頁面
2. 聯絡系統管理員查詢權限清單
3. 嘗試訪問功能時系統會顯示權限狀態
4. 查看用戶手冊了解角色權限範圍

### Q: 權限被誤刪或修改怎麼處理？

**A:** 建議採取以下步驟：
1. 立即聯絡系統管理員
2. 說明權限變更的具體情況
3. 提供權限變更前的使用記錄
4. 等待管理員調查和恢復權限
5. 必要時可申請臨時權限
